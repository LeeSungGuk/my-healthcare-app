# Development History

This folder records product, design, and implementation decisions that are useful
to recover later.

Use these notes for context that does not fit cleanly in a commit message:

- What problem was being solved
- Which options were considered
- Why a direction was chosen
- What changed in the product or code
- How the change was verified
- What remains unresolved

Git history answers what changed. These notes answer why it changed.

## Folder Structure

History notes are grouped by month.

```text
docs/history/
├── README.md
└── YYYY-MM/
    └── YYYY-MM-DD-topic.md
```

Use the month folder for every dated history note. Keep only this README at the
`docs/history/` root.
